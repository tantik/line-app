# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ)
## LINE Mini App SaaS для онлайн-записи в салоны красоты

**Версия:** 1.0  
**Дата:** 28 апреля 2026 г.  
**Статус:** Активная разработка  
**Цель:** Создание демо-версии SaaS продукта с лидогенерацией для демонстрации клиентам

---

## 1. ВВЕДЕНИЕ

### 1.1 Описание продукта
LINE Mini App SaaS - это облачная платформа для автоматизации онлайн-записи в салоны красоты и аналогичные бизнесы в Японии. Продукт позволяет салонам создавать собственные мини-приложения в LINE для приема онлайн-бронирований без необходимости разработки.

**Ключевые ценности:**
- **Для салонов:** Автоматизация записи, снижение no-show, улучшение клиентского сервиса
- **Для клиентов:** Удобная запись через LINE, напоминания, подтверждения
- **Для владельца продукта:** SaaS с подпиской, масштабируемость на 1-5 клиентов на старте

### 1.2 Цели проекта
1. Создать полностью рабочую демо-версию для презентаций клиентам
2. Реализовать лидогенерацию через демо-сайт
3. Обеспечить премиальное качество UX/UI
4. Подготовить фундамент для коммерческой версии

## 2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### 2.1 Демо-приложение для записи (Основной продукт)

#### 2.1.1 Экран 1/3: Выбор услуги и мастера
- **Отображение услуг:** Горизонтальная прокрутка карточек услуг с иконками
- **Фильтрация мастеров:** При выборе услуги показываются только мастера, которые ее выполняют
- **Выбор мастера:** Опционально, можно выбрать услугу без мастера
- **Параллельная работа:** Если на одно время работают несколько мастеров - они показываются как доступные

#### 2.1.2 Экран 2/3: Выбор даты и времени
- **Календарь:** Показ дат на 14 дней вперед
- **Временные слоты:** Динамическое отображение доступного времени
- **Фильтрация по мастеру:** Если мастер выбран - показывается его расписание
- **Занятые слоты:** Неактивные/скрытые для занятого времени
- **Свободные мастера:** Внизу экрана показываются все свободные мастера на выбранное время
- **Выделение выбранного мастера:** Активный мастер выделен

#### 2.1.3 Экран 3/3: Подтверждение заказа
- **Сводка заказа:** Услуга, мастер, дата/время, цена
- **Ввод данных:** Имя и телефон (обязательно)
- **Валидация:** Попап/модалка при отсутствии данных
- **Подтверждение:** Кнопка "確定" с отправкой в LINE

#### 2.1.4 Экран "Спасибо"
- Подтверждение успешного создания
- Инструкция о получении сообщения в LINE

### 2.2 LINE интеграция

#### 2.2.1 Подтверждение заказа
- Автоматическое отправка сообщения в LINE после бронирования
- Содержит детали заказа + кнопки подтверждения/отмены

#### 2.2.2 Напоминания
- За 24 часа до услуги
- За 3 часа до услуги (но не ранее 9:00 утра)
- Через LINE Messaging API

#### 2.2.3 Управление статусом
- Кнопки в LINE: Подтвердить/Отменить
- Обновление статуса в системе

### 2.3 Anti No-Show система

#### 2.3.1 Risk Status
- Автоматическое определение рискованных бронирований
- Risk score на основе истории клиента
- Пометка "risk" за N минут до начала

#### 2.3.2 Механизмы предотвращения
- Напоминания в LINE
- История отмен/неявок
- Блокировка проблемных клиентов (будущая функция)

### 2.4 Лидогенерация

#### 2.4.1 Демо-сайт
- Краткое описание продукта и преимуществ
- Форма обратной связи
- Демо-админка без регистрации

#### 2.4.2 Сбор лидов
- Данные с сайта
- Данные из демо-приложения
- Статус лида, заметки, источник

### 2.5 Админ-панель

#### 2.5.1 Управление услугами
- CRUD услуги: название, код, категория, длительность, цена, иконка
- Сортировка отображения

#### 2.5.2 Управление персоналом
- CRUD мастера: имя, фото, время работы, выходные дни
- Связи мастер-услуга (многие-ко-многим)

#### 2.5.3 Управление записями
- Просмотр всех бронирований
- Изменение статуса
- Блокировка времени

#### 2.5.4 Супер-админка (для владельца)
- Контроль всех салонов
- Управление лидами
- Мониторинг состояния продукта

## 3. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### 3.1 Архитектура
- **Frontend:** HTML/CSS/JS (Vanilla), Vercel hosting
- **Backend:** Supabase (PostgreSQL + Auth + Storage + Functions)
- **LINE интеграция:** LIFF SDK, Messaging API
- **Мульти-тенантность:** RLS политики для изоляции данных салонов

### 3.2 База данных (Supabase PostgreSQL)

#### 3.2.1 Основные таблицы
- `salons` - салоны
- `salon_members` - пользователи салона
- `staff` - мастера (с work_days, start_time, end_time)
- `services` - услуги (с icon_url)
- `staff_service_map` - связи мастер-услуга
- `bookings` - бронирования
- `blocked_slots` - заблокированное время
- `reminder_jobs` - очередь напоминаний
- `leads` - лиды с сайта/демо

#### 3.2.2 RPC функции
- `available_slots_v2` - расчет доступных слотов
- `create_public_booking` - создание бронирования
- `public_change_booking_status` - изменение статуса
- `mark_booking_risk` - определение рисков

### 3.3 Безопасность
- **Аутентификация:** Supabase Auth (email OTP для админов)
- **Авторизация:** RLS политики
- **Конфиденциальность:** Шифрование чувствительных данных
- **API безопасность:** Service role key для серверных функций
- **Валидация:** Проверка всех входных данных

### 3.4 Производительность
- **Кеширование:** Слотов на клиенте
- **Оптимизация:** Индексы на часто используемые поля
- **Масштабируемость:** Serverless архитектура
